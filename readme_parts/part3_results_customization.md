## 结果解读指南

### 输出文件概述

计算完成后会生成 `allindex.xlsx`，包含 **8个工作表**，分别对应不同的效率指标和分解方式。

### 工作表1：sbmeffs（基础效率值）

这是所有计算的原始效率值，包含10列：

| 列名 | 含义 |
|------|------|
| dmu | 决策单元名称 |
| 时间 | 年份/时期 |
| ectt | CRS下的当期效率 E_t(x_t,y_t) |
| evtt | VRS下的当期效率 |
| ectt1 | CRS下的跨期效率 E_t(x_{t+1},y_{t+1}) |
| evtt1 | VRS下的跨期效率 |
| ect1t | CRS下的反向跨期效率 E_{t+1}(x_t,y_t) |
| evt1t | VRS下的反向跨期效率 |
| ecgt | CRS下的全局效率 E_g(x_t,y_t) |
| evgt | VRS下的全局效率 |

**效率值判断标准**：

| 效率值 | 含义 | 说明 |
|--------|------|------|
| < 1 | 无效 | 存在改进空间 |
| = 1 | 有效 | 处于效率前沿面上 |
| > 1 | 超效率 | 超越了效率前沿（仅超效率模式下出现） |

**CRS vs VRS 的区别**：
- **CRS**（常规模报酬）：假设所有单元都在最优规模运营，适合宏观比较
- **VRS**（可变规模报酬）：允许不同规模运营，排除了规模效率的影响，更适合微观分析

### 工作表2：FGLR1992_C（Malmquist指数 - CRS分解）

这是最常用的分解结果之一：

| 指标 | 含义 | >1表示 | <1表示 |
|------|------|--------|--------|
| mlc(M指数) | Malmquist生产率指数 | 生产率提升 | 生产率下降 |
| ecc(效率变化) | 技术效率变化 | 追赶效应增强 | 追赶效应减弱 |
| tcc(技术进步) | 技术进步变化 | 技术前沿外推 | 技术前沿内缩 |

**解读示例**：
- `MLC = 1.05, ECC = 1.02, TCC = 1.03` → 生产率提升5%，其中效率提升2%，技术进步3%
- `MLC = 0.95, ECC = 0.98, TCC = 0.97` → 生产率下降5%，效率和技改都在退步

### 工作表7：PL2005_C（Global Malmquist - CRS）

引入全局前沿的Malmquist指数：

| 指标 | 含义 | >1表示 |
|------|------|--------|
| mgc(GM指数) | 全局Malmquist指数 | 全局生产率提升 |
| ecc(效率变化) | 效率变化 | 效率提升 |
| bpcc(技术差距变动) | 当期前沿与全局前沿的差距变化 | 向全局前沿靠拢 |

---

### 工作表8：PL2005_V（Global Malmquist - VRS）

| 指标 | 含义 |
|------|------|
| mgv(GM指数) | VRS下的全局Malmquist指数 |
| ecv(效率变化) | VRS下的效率变化 |
| bpcv(技术差距变动) | VRS下的技术差距变动 |

> 💡 **全局Malmquist vs 传统Malmquist**：传统ML指数只比较相邻两期，可能出现"技术退步"的矛盾结果；全局ML以所有时期数据构建统一前沿，避免了这一问题。

---

### 关键概念速查

| 概念 | 说明 |
|------|------|
| Malmquist指数 >1 | 生产率提升 |
| Malmquist指数 =1 | 生产率不变 |
| Malmquist指数 <1 | 生产率下降 |
| 效率变化（EC） | 反映决策单元"追赶"前沿的能力 |
| 技术进步（TC） | 反映前沿本身的移动 |
| 全局前沿 | 用所有时期数据构建的统一参考前沿 |
| 技术差距（BPC） | 当期前沿与全局前沿的差距 |

**判断生产率变动的驱动因素**：
- `EC > 1 且 TC > 1` → 效率提升和技术进步共同驱动
- `EC > 1 且 TC < 1` → 仅靠效率追赶，技术退步
- `EC < 1 且 TC > 1` → 技术进步但效率下滑
- `EC < 1 且 TC < 1` → 双重恶化

## 自定义修改指南

### 直接修改参数

打开 `dea_calculator.py`，找到第640-646行，直接修改参数值：

```python
dmus = 30       # 决策单元数
periods = 5     # 观察时期数
nx = 3          # 投入变量个数
ny = 4          # 产出变量个数
nb = 1          # 非期望产出变量个数
undesirable = 1 # 1=有非期望产出, 0=无
sup = 1         # 1=超效率, 0=标准效率
```

> 💡 推荐使用 `runner.py` 交互式输入，无需手动修改代码！

### 添加新的投入/产出变量

**步骤**：
1. 在 `t1.xlsx` 中添加新的列（注意列顺序！）
2. 修改对应的参数值（`nx`、`ny` 或 `nb`）
3. 确保新增列的数据类型为数值且无缺失值

### 主要函数功能列表

| 函数名 | 功能 |
|--------|------|
| `Ecv_tt` | 计算当期SBM效率 E_t(x_t,y_t) |
| `Ecv_tt1` | 计算跨期效率 E_t(x_{t+1},y_{t+1}) |
| `Ecv_t1t` | 计算反向跨期效率 E_{t+1}(x_t,y_t) |
| `SupEcv_tt` | 超效率SBM计算 |
| `SupEcv_tt1` | 超效率跨期计算 |
| `SupEcv_t1t` | 超效率反向跨期计算 |
| `Ec_g` | 全局效率计算 |
| `SupEc_g` | 超效率全局计算 |

### 注意事项

1. **备份原文件**：修改代码前务必备份 `dea_calculator.py`
2. **先用小数据测试**：修改后先用3-5个DMU的小数据集验证
3. **列索引必须正确**：代码中通过列索引读取数据，列顺序错误会导致结果完全错误
4. **超效率可能无可行解**：某些情况下超效率模型无解，程序会自动回退到标准效率
